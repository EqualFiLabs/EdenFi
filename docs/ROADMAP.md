# EdenFi Roadmap

Near-term direction:

1. Define the minimum reusable substrate to extract from EqualFi.
2. Decide which EDEN primitives should migrate onto that substrate first.
3. Port shared storage, position, encumbrance, and fee-routing primitives.
4. Rebuild EDEN-specific product logic on top of the extracted core.

Working principle:

EDEN should be the first launchable product built on the EdenFi substrate, not
a separate architecture that later needs to be discarded.
