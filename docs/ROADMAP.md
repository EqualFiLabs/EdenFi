# EqualFi Substrate Roadmap

Near-term direction:

1. Define the minimum reusable substrate to extract from EqualFi.
2. Decide which EDEN primitives should migrate onto that substrate first.
3. Port shared storage, position, encumbrance, and fee-routing primitives.
4. Rebuild EDEN by EqualFi product logic on top of the extracted core.

Working principle:

EqualFi is the substrate.

EDEN by EqualFi should be the first launchable product built on that substrate,
not a separate architecture that later needs to be discarded.
